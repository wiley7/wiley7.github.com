cocos2d-x 保持游戏不退出

背景

iOS应用的几个状态

参考冰萧的博文iOS应用程序状态切换相关
状态如下：

  1. Not running 没有运行
  2. Inactive 当前应用正在前台运行，但是并不接收事件
  3. Active 前台运行
  4. Background 后台运行
  5. Suspended 后台挂起

状态切换的几个方法

  1. applicationDidEnterBackground 此时应用处在background状态，并且没有执行任何代码，未来将被挂起进入suspended状态。
  2. applicationWillEnterForeground 当前应用正从后台移入前台运行状态，但是当前还没有到Active状态时执行的函数。
  3. application:didFinishLaunchingWithOptions 这是程序启动时调用的函数。
  4. applicationDidBecomeActive 应用在准备进入前台运行时执行的函数。
  5. applicationWillResignActive 应用当前正要从前台运行状态离开时执行的函数。

保持应用后台挂起的可能问题

  1. 数据不同步
  2. 某些正在执行的action出错
  3. Scheduler的问题

####1. 数据不同步
在游戏处于中断状态时，我们的游戏中体力、精力等会随时间发生变化，其他一些与时间相关的数据，如某些倒计时等，等会处于暂停状态，在游戏恢复之后，会与后台不同步

####2. 正在执行的action出错
当游戏处于background时，如果仍然在调用opengl绘制，会报错

####3. Scheduler的问题
跟1类似，主要是执行次数的问题。如一个操作，每隔一段时间执行一次，当处于background时，会暂停，但恢复之后，接着上次的继续运行，会让人感觉中间有个明显的暂停

保持应用不退出

app的应用配置在info.plist中，其中有一项是Application does not run in background 将其设为NO，在应用退出到background时，可以保持suspended状态

关于Scheduler

cocos2d-x 自己的CCScheduler，使用的是间隔时间。
如：一个操作，每个10秒执行一次。当执行一次过了4秒之后，按home退出到后台，隔7秒重进，此时CCScheduler会继续等待6秒中之后才去执行第二次。
新增一个PCScheduler，实现根据精确时间来确定是否执行。上述例子，对于PCScheduler,就直接执行一次，无需等待。
对于我们的某些情况，比如倒计时，需要使用第二种Scheduler，让玩家感受不到中间是暂停的
