top命令显示

一般一个top的显示结果如下：

top - 20:53:44 up 12 days, 3:53, 1 user, load average: 0.00, 0.01, 0.00
Tasks: 398 total, 1 running, 397 sleeping, 0 stopped, 0 zombie
Cpu(s): 0.1%us, 0.0%sy, 0.0%ni, 99.8%id, 0.1%wa, 0.0%hi, 0.0%si, 0.0%st
Mem: 8181612k total, 3408752k used, 4772860k free, 182508k buffers
Swap: 8388600k total, 12k used, 8388588k free, 2020072k cached
PID USER PR NI VIRT RES SHR S %CPU %MEM TIME+ COMMAND
9583 icyleaf 15 0 13012 1536 952 R 0.3 0.0 0:00.13 top

之前一直没有仔细看top命令的显示内容。最近在对服务器进行压力测试，因此开始使用它来实时观测服务器的负载情况。
主要了解了以下几点：

  • load average 负载综合情况，一般认为值小于CPU的核数，都是在允许范围内。
  • CPU的几个数值
    • us 用户进程的CPU占用时间
    • sy 系统进程的CPU占用时间
    • ni 用户进程空间内改变过优先级的进程占用CPU时间
    • id CPU空闲的时间
    • wa CPU等待硬盘IO的时间
    • hi 硬中断的CPU占用时间
    • si 软中断的CPU占用时间
  • Mem内存的分配情况
  • 后面的列表是各进程的情况
